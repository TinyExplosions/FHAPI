$fh.act is the main entrypoint to the cloud from a FeedHenry hybrid app. It allows you to call cloud side actions and perform requests to the cloud. In this sample application, we have also defined App.ajaxManager.mayAct which is a wrapper that allows calls to be queued for later sending if the device is offline.

Use the buttons below to demostrate the difference in behaviour. You will see results in the console.

A 'standard' $fh.act call

$fh.act({
  act: "testAct",
  req: {
    attributes: options
  }
  }, function(res){
    console.log("Returned From Server");
  }, function(err){
    console.log("WAT!?");
});
 $fh.act

mayAct -button presses will be queued and sent whan app is online, even if you navigate away from the page (use the 'Connection' button to fake online/offline states
